defer & async

<script defer src="script.js"></script>  or  <script src="script.js" defer></script>
<script async src="script.js"></script>  or  <script src="script.js" async></script>

  • defer 속성을 사용하면 HTML Parser가 문서를 파싱하는 동안 JavaScript 코드가 백그라운드에서 로드된다. 이 경우 문서의 로딩 시간이 더 빨라진다.
    또한 이 속성을 사용하면 문서가 파싱(로딩)되는 동안은 JavaScript 코드가 실행되지 않고 파싱이 완료된 후 실행된다.
  • async는 defer과 다르게 js script 다운로드가 완료되면 즉시 실행된다. async를 사용하면 스크립트가 실행되는 순서가 보장되지 않으므로
    DOM 부분을 조작하는 스크립트에서는 사용을 지양해야 한다.

DOM Element & querySelector

item이 셀렉터 문자열이어도 자동으로 변환해줌 / item이 진짜 엘리먼트가 아닐 경우 에러 대응

function elemSetting(item) {
	// 1. item이 문자열이면 querySelector로 찾는다
	if (typeof item === 'string') {
		item = document.querySelector(item);
	}

	// 2. item이 없거나 querySelector가 없으면 경고 후 종료
	if (!item || typeof item.querySelector !== 'function') {
		console.warn('Invalid item passed to elemSetting:', item);
		return;
	}

	// 3. .elemChild를 찾는다
	const elemChild = item.querySelector('.elemChild');

	if (elemChild) {
		console.log('aa');
	} else {
		console.warn('No .elemChild found inside item:', item);
	}
}

sessionStorage, localStorage

브라우저에 키, 값을 저장. (이 저장한 값을 이용해서 새 창 열기할 때 이전 페이지와 연관된 자바스크립트를 실행할 수 있다.)
- sessionStorage: 오리진이 같은 브라우저 탭, iframe에서 공유. 페이지를 새로 고침 해도 남음. 탭이나 브라우저를 종료하면 사라짐.
- localStorage: 오리진이 같은 탭, 창 전체에서 공유, 브라우저를 껐다 켜도 남아있다. 

// ex.1 : page01의 버튼 클릭시, page02가 새 창에서 열리면서 이벤트를 바로 실행할 수 있게 함
// page01의 스크립트
$('.btn').on('click', function () {
	const btntxt = $(this).text();
	if ( btntxt == 'example 1' ) {
		var name = 'remember_example'
		localStorage.setItem('name', name)
	}
})

// page02의 스크립트
var loadName = localStorage.getItem('name')
.... script 작성....
if ( loadName == 'example 1' ) {
 ... 실행할 스크립트 ...
 localStorage.clear();
}
